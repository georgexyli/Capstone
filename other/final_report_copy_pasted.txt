Group Highlights

Our project focused on the design and implementation of a decentralized finance (DeFi) AI agent that allows users to execute blockchain transactions using natural language. Instead of interacting directly with complex wallet interfaces and complicated protocols, users can perform actions simply by typing a short command. The goal was to make decentralized finance more accessible while still keeping the system secure.

Our final system successfully integrates a Telegram frontend interface with a modular backend execution pipeline. A user can send a message, such as a swap request, and the system will correctly interpret the command, convert it into a structured transaction, and guide the user throughout the execution process. The pipeline includes intent parsing, routing, simulation, human confirmation, signing, and transaction broadcasting, which ensures that each step is handled in a clear and controlled way.

One of the main achievements of our project is the implementation of a safety focused, non-custodial design. Every transaction requires explicit confirmation from the user before it is executed, and private keys are never stored on a public server. In addition, the system performs a simulation step before execution. This allows invalid transactions or incorrect inputs to be detected and stopped early, and prevents any transactions from exceeding user defined slippage or gas limits. This makes the system safer and easier to use, especially for users who are not familiar with blockchain tools.

The system was successfully tested on the Ethereum Sepolia test network and demonstrated reliable end-to-end functionality. The agent was able to consistently interpret user commands, generate valid transaction data, execute token swaps, and receive confirmed transaction results on-chain. These results validate that the system works as intended and that the integration between the natural language interface and the blockchain execution pipeline is correct.

From a technical standpoint, the project also demonstrates the effectiveness of a modular and extensible architecture. By building on top of the Spartan framework and focusing on the interaction layer, the team was able to concentrate on usability, safety, and system integration rather than rebuilding low-level blockchain components from scratch.

Overall, the final prototype shows that natural language interfaces can drastically reduce the difficulty of interacting with decentralized finance systems while still maintaining user control and security. The project successfully meets its core objectives and provides a strong foundation for future improvements and further development.

Executive Summary

This project presents the design and implementation of a Decentralized Finance (DeFi) AI Agent that allows users to execute blockchain transactions through simple natural language commands. The system aims to reduce the complexity of interacting with DeFi protocols by abstracting low-level technical details such as smart contracts, gas fees, and transaction construction, while still maintaining user control and security.

The main idea of the system is to act as a transaction execution assistant instead of an autonomous decision-maker. Users specify their intent in plain language (e.g., swapping one token for another one), then the system translates this input into structured instructions that are executed on blockchain.

The system architecture is built around a pipeline that processes the user input through stages: natural language parsing, routing and quote generation, human confirmation, simulation and risk checking, transaction construction, signing, and broadcasting. The implementation utilizes a forked version of the Spartan execution framework, extended to support Ethereum-based transactions on the Sepolia test network. A Telegram-based interface was developed to provide a lightweight and accessible frontend for user interaction.

Testing and verification results show that our system successfully meets its core functional objectives. The agent can correctly interpret natural language commands, generate human-readable transaction summaries, and execute valid swaps on the blockchain. Transactions were verified through on-chain confirmation, showing the correctness of execution. The system also provides important safety features, including explicit confirmation before execution and rejection of invalid or unrealistic inputs with clear error messages.

While the system performs well within its intended scope, a few requirements were only partially validated, and one feature, slippage tolerance enforcement, was not implemented. These limitations highlight areas for future improvement but do not impact the core functionality of the system.

From a design perspective, the project demonstrates that combining a natural language interface with a modular, non-custodial execution pipeline is a practical and effective approach for improving accessibility in DeFi. Our use of a gated execution flow ensures that users will remain in control of all actions that happen, while the integration of simulation and validation steps enhance safety and usability.

In summary, this project delivers a working prototype of a DeFi AI agent that makes blockchain interaction easier while preserving security and user control. The results show that these types of systems are feasible and valuable, providing a foundation for future development in AI-assisted decentralized applications.


Final Design

System-Level Overview


Module-Level Descriptions and Designs

The system is built as a series of modules that each handle one part of the pipeline from user input to transaction execution. Instead of trying to do everything in one place, we split the logic into smaller components so that each piece is easier to understand, test, and modify.

The first module is the natural language parser. This is where the user’s message is processed. A command like “swap 0.1 ETH to USDC” gets converted into a structured format with fields such as action, token pair, and amount. We used an LLM here since user input can vary a lot in wording. The parser also handles invalid inputs and returns clear error messages when something is missing or unclear.

Next is the routing and execution planning module. Once we have a structured intent, this module decides how to actually carry it out. It maps the request to a specific operation, such as a Uniswap swap, and prepares the required parameters. This part also connects to the reused Spartan execution code, which handles the lower-level interaction with the blockchain.

Before anything is executed, the request goes through the simulation and validation module. This step checks whether the transaction is likely to succeed. It looks at things like available balance and basic correctness of the request. If something is wrong, the system stops here and explains the issue to the user. This prevents unnecessary failed transactions.

After that, the confirmation module presents a summary of the transaction to the user. This includes what is being swapped and any relevant details such as estimated cost. The user has to explicitly confirm before the system continues. Without this step, nothing gets executed.

The signing module handles transaction authorization. Signing is done locally using the user’s wallet. Private keys are never sent to the backend, which keeps the system non-custodial. If a valid signer is not available, the process does not continue.

Once signed, the transaction is passed to the execution module. This module sends the transaction to the Ethereum Sepolia network and returns a transaction hash. The user can then verify the result on-chain.

Finally, there is a logging module that records each step in the pipeline. This includes parsing, routing, simulation, confirmation, signing, and sending. The logs are useful for debugging and verifying system behavior. Sensitive information such as private keys is never stored.

Overall, this modular setup made it much easier to build and test the system. Each part could be worked on separately, and issues were easier to track down when something went wrong.

Assessment of Final Design

The final design meets the main goal of the project, which was to make blockchain transactions easier to use without taking control away from the user. The system works end to end and shows that it is possible to go from a simple text command to a confirmed on-chain transaction.

One of the strongest parts of the design is the focus on safety. The system never executes anything automatically. Every transaction goes through a confirmation step, and the user has to approve it. The simulation step also helps catch obvious issues before they reach the blockchain. This is important because mistakes in blockchain transactions cannot be undone.

Another good design choice was keeping everything non-custodial. Private keys stay with the user, and the system never handles them directly. This reduces risk and makes the system more trustworthy. It also aligns with how most modern DeFi tools are designed.

The modular structure also worked well. Breaking the system into separate components made development more manageable. It also makes future improvements easier. For example, the parser or simulation module could be improved without changing the rest of the system.

That said, there are still some limitations. The simulation step is fairly basic and does not fully handle things like slippage control. This means one of the original requirements was not fully implemented. Testing was also somewhat limited. While the system worked in the cases we tried, it was not evaluated under heavy usage or a wide range of edge cases.

The choice of a Telegram interface made the system easy to use and quick to build, but it also limits how much information can be shown to the user. A more advanced interface could make the system clearer and easier to interact with, especially for more complex transactions.

Overall, the design works well as a prototype. It shows that the core idea is viable and that the system can be built in a clean and modular way. There is still room for improvement, but the main concepts were validated and the system performs as expected in its current scope.

Summary and Conclusion:

This project was set to design and implement a decentralized finance (DeFi) AI agent that allows users to execute blockchain transactions through a natural language interface. The core objective was not to automate trading decisions, but to simplify and safely mediate transaction execution by translating user intent into valid blockchain operations. The system focused on accessibility, security, and controlled execution, particularly for users with limited experience interacting directly with smart contracts.

Overall, the system successfully achieved its primary design goals. Through testing and verification, we showed that the agent can correctly interpret natural language commands, generate valid transaction data, and execute swaps on the Ethereum Sepolia test network. The integration of a Telegram-based interface allowed for a lightweight and accessible user experience, while the backend execution pipeline ensured that all actions followed a structured and gated process. Key requirements such as human confirmation before execution, client-side signing, and non-autonomous behavior were implemented and validated through test cases.

The testing and verification results present strong evidence that the final system meets the core functional requirements. In particular, the system demonstrated reliable performance in parsing user input into structured intents, generating swap previews, and enforcing confirmation gates prior to execution. Successful transactions were executed on Sepolia and verified via Etherscan, confirming the correctness of the transaction construction and broadcasting pipeline. Additionally, the system handled invalid inputs with clear error messages, which supports the usability and robustness of the interface.

While many core requirements were met, some were only partially validated due to time and scope constraints. For example, gas estimation and latency requirements were demonstrated on limited test cases rather than large-scale benchmarks. The slippage tolerance requirement was not implemented, and therefore remains untested. These limitations indicate that while the system is functionally complete for demonstration purposes, further work is needed to reach production-level robustness.

Despite these limitations, the key design ideas of the project were validated. The agent-based architecture built on top of the Spartan framework proved to be an effective approach for combining modularity with execution reliability. By leveraging an existing execution engine and extending it with a natural language interface, our team focused on higher-level interaction and safety mechanisms instead of rebuilding blockchain infrastructure from scratch. The use of a gated execution flow- parse, route, confirm, simulate, sign, and broadcast- ensured that user control and safety remained essential throughout the process. This design aligns well with the project’s goal of staying away from autonomous trading while still improving accessibility.

One of the most important conclusions from this project is that natural language interfaces can significantly reduce the barrier to entry for interacting with decentralized finance systems. Traditional DeFi interactions require users to understand wallets, gas fees, smart contracts, and transaction data, which can be intimidating and error-prone. By moving these complexities behind a structured chatbot interface, the system provides a viable path toward more user-friendly DeFi applications. At the same time, the inclusion of explicit confirmation steps and simulation checks highlights the importance of maintaining transparency and user control in financial systems.

From an industry perspective, this type of system is relevant to ongoing developments in AI-assisted financial tools and decentralized applications. Many current platforms are exploring ways to integrate large language models with blockchain systems, particularly for user interaction and automation [1]. The approach taken in this project- combining a language interface with a non-custodial, user-controlled execution pipeline, aligns with emerging best practices in both AI safety and decentralized system design. In academia, this work contributes to the growing area of research at the intersection of natural language processing, agent-based systems, and blockchain technologies [1].

There are many directions for future work that could significantly enhance the system. First, implementing a full simulation and risk engine with precise slippage enforcement would improve transaction safety and bring the system closer to real-world deployment standards. Second, expanding support for multiple protocols and chains would increase flexibility and applicability of the agent. Third, maybe integrating a graphical user interface alongside the Telegram bot could provide better visibility into transaction details and system state. Also, more comprehensive performance testing, including latency benchmarking and stress testing, would be needed to validate the system under stricter and more realistic conditions.

Another important area for improvement is security and user experience. While the system we did follows a non-custodial model, further work could include integration with hardware wallets or secure signing modules to enhance key protection. On the user experience side, improving error messaging, handling edge cases more gracefully, and providing clearer explanations of transaction risks would make the system more accessible to beginners.

In conclusion, the project successfully demonstrates a functional prototype of a decentralized finance AI agent that bridges natural language interaction with blockchain execution. While not all requirements were fully implemented or validated, the system meets its core objectives and provides a strong foundation for future development. The results show that it is both feasible and valuable to combine AI-driven interfaces with decentralized systems in a way that preserves user control and enhances accessibility.
